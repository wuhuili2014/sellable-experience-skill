# Codex Skill 发布与安装说明

对应 Skill：`sellable-experience-skill`

适用产品：《把你的经验做成可售卖产品》

## 1. 你需要发布到 GitHub 的内容

仓库里至少保留这个结构：

```text
your-repo/
└── sellable-experience-skill/
    └── SKILL.md
```

其中 `SKILL.md` 顶部必须包含：

```markdown
---
name: sellable-experience-skill
description: 当用户想做课、做知识产品、副业、转型、自由职业，或不知道自己有什么经验可以卖时，帮助用户从经历中找到一个可验证的知识产品方向，并给出低交付压力的下一步验证动作。
---
```

## 2. 用户如何让 Agent 安装

你可以在产品交付页里这样写：

如果你使用 Codex，可以直接对你的 Agent 说：

```text
请帮我安装这个 Skill：
https://github.com/你的用户名/你的仓库/tree/main/sellable-experience-skill
```

安装完成后，重启 Codex。

之后你可以这样启动：

```text
用可售卖经验定位 Skill 帮我找到一个适合做成知识产品的方向。
```

也可以直接说：

```text
我想做知识产品，但不知道自己有什么经验可以卖，你一步步问我。
```

## 3. 公开仓库和私有仓库

建议先用公开仓库发布，用户安装阻力最低。

如果用私有仓库，用户需要自己的 Agent 能访问该仓库，或者本地已经配置好 GitHub 权限。

## 4. 面向普通用户的说法

不要把它包装成“下载安装一个技术文件”。

可以这样说：

这是一个可以安装到 Codex 里的 AI 小助手。它会通过提问和整理，帮你从自己的经历里找到一个可验证的知识产品方向，并给你今天就能做的需求验证动作。

如果你不用 Codex，也可以使用后续提供的“复制即用提示词版”。

## 5. 发布前检查

- GitHub 链接是否能打开到 `sellable-experience-skill` 文件夹？
- 文件夹里是否有 `SKILL.md`？
- `SKILL.md` 顶部是否有 `name` 和 `description`？
- 安装后是否提醒用户重启 Codex？
- 触发语是否足够自然，不要求用户记住 Skill 英文名？
