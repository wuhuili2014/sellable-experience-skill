# Agent Skill 发布与安装说明

对应 Skill：`sellable-experience-skill`

适用产品：《把你的经验做成可售卖产品》

## 1. 你需要发布到 GitHub 的内容

仓库里至少保留这个结构：

```text
your-repo/
└── sellable-experience-skill/
    └── SKILL.md
```

其中 `SKILL.md` 顶部必须包含：

```markdown
---
name: sellable-experience-skill
description: 当用户想做课、做知识产品、副业、转型、自由职业，或不知道自己有什么经验可以卖时，帮助用户从经历中找到一个可验证的知识产品方向，并给出低交付压力的下一步验证动作。
---
```

## 2. 用户如何让 Agent 安装

你可以在产品交付页里这样写：

如果你使用 Codex，可以直接对你的 Agent 说：

```text
请帮我安装这个 Skill：
https://github.com/你的用户名/你的仓库/tree/main/sellable-experience-skill
```

安装完成后，重启 Codex。

之后你可以这样启动：

```text
用可售卖经验定位 Skill 帮我找到一个适合做成知识产品的方向。
```

也可以直接说：

```text
我想做知识产品，但不知道自己有什么经验可以卖，你一步步问我。
```

## 3. 公开仓库和私有仓库

建议先用公开仓库发布，用户安装阻力最低。

如果用私有仓库，用户需要自己的 Agent 能访问该仓库，或者本地已经配置好 GitHub 权限。

## 4. 面向普通用户的说法

不要把它包装成“下载安装一个技术文件”。

可以这样说：

这是一个可以安装到 Codex 里的 AI 小助手。它会通过提问和整理，帮你从自己的经历里找到一个可验证的知识产品方向，并给你今天就能做的需求验证动作。

如果你不用 Codex，也可以使用后续提供的“复制即用提示词版”。

## 5. 发布前检查

- GitHub 链接是否能打开到 `sellable-experience-skill` 文件夹？
- 文件夹里是否有 `SKILL.md`？
- `SKILL.md` 顶部是否有 `name` 和 `description`？
- 安装后是否提醒用户重启 Codex？
- 触发语是否足够自然，不要求用户记住 Skill 英文名？

## 6. 其他 Agent 的安装方式

这个 Skill 使用的是通用 Agent Skills 结构：一个文件夹里包含 `SKILL.md`。因此只要某个 Agent 支持 Agent Skills / Custom Skills，就可以复用这份内容。

### Claude Code

Claude Code 支持本地 Custom Skills。用户可以把文件夹放到：

```text
~/.claude/skills/sellable-experience-skill/SKILL.md
```

也可以放到某个项目内：

```text
.claude/skills/sellable-experience-skill/SKILL.md
```

安装后可以用自然语言触发，也可以用技能名触发。

### Claude 网页版

Claude 网页版支持上传自定义 Skill。用户可以：

1. 下载或打包 `sellable-experience-skill` 文件夹为 ZIP。
2. 进入 Claude 的 Customize > Skills。
3. 选择 Create skill / Upload a skill。
4. 上传 ZIP。
5. 启用这个 Skill。

### WorkBuddy

WorkBuddy 如果开启了 Skills 功能，可以尝试使用下面两个目录：

```text
~/.workbuddy/skills/sellable-experience-skill/SKILL.md
```

或项目级目录：

```text
.workbuddy/skills/sellable-experience-skill/SKILL.md
```

但 WorkBuddy 的具体安装入口可能会随版本变化，建议在产品说明里写成：

如果你使用 WorkBuddy，可以把这个 Skill 文件夹安装到 WorkBuddy 的 Skills 目录；如果你的版本支持从 GitHub 安装 Skill，也可以直接使用 GitHub 链接。

### GitHub CLI

如果用户安装了新版 GitHub CLI，并且本地 Agent Host 支持 `gh skill`，可以尝试：

```text
gh skill install wuhuili2014/sellable-experience-skill sellable-experience-skill
```

不同 Agent 的目标目录可能不同，用户可以根据自己的 Agent 类型指定安装目标。

## 7. 对外推荐说法

建议你对外不要只说“Codex Skill”，而是说：

这是一个符合 Agent Skills 格式的 AI 小助手，支持 Codex、Claude Code，以及其他兼容 Agent Skills 的工具。

然后在交付页分成三种使用方式：

1. Codex 用户：直接让 Agent 从 GitHub 安装。
2. Claude 用户：Claude Code 放入 skills 目录；Claude 网页版上传 ZIP。
3. 普通用户：使用复制即用提示词版。
